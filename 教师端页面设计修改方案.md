# 教师端页面设计修改方案

## 0. 需求总览与背景

- **需求来源文件**：阅读并整合 `1. 设计与规划.md`、`2.项目原则.md`、`3.项目完整文件结构清单.md`、以及过往《教师端页面一致性规划》《教师端页面设计文档》内容，对齐当前教师端代码实现（`apps/web-teacher/src`）。
- **平台定位**：ReOpenInnoLab 面向中小学开放创新教育，将 AI 与信息技术课程作为特色亮点，同时保持多学科教学支撑能力。
- **用户视角**：以人工智能/信息技术教师为核心角色设计体验，但语义与组件需支持其他学科便捷复用，保障备课、课堂、实验、作业、洞察全链路畅通。
- **权限与数据**：教师登录后仅能访问本人负责的课程/班级/实验，支持多学科标签切换，严控展示范围；默认视图聚焦“我的学科”。
- **体验基调**：Dashboard 聚焦 AI/IT 指标并提供学科筛选；课程与实验模块突出 AI/编程场景，同时保留通用模板；AI 助手与工具围绕 AI/IT 深耕，支持提示词/模板插拔复用。
- **导航重构方向**：侧边栏需突出教师高频任务（课程、实验管理、作业、学习分析、AI 助手），全局搜索收敛为“我的课程搜索”或迁移至顶栏，与 `AppSidebar.vue` 当前实现对齐调整。

## 1. 目标与设计原则
- **降低信息密度**：剔除重复模块，优化布局层级，让教师在首屏快速抓住重点。
- **强化任务导向**：围绕“备课 - 上课 - 作业 - 分析”主链路组织内容，减少无关干扰。
- **构建可扩展框架**：统一组件与布局语义，方便后续新增模块时保持一致体验。
- **保持温暖科技调性**：延续原有玻璃质感与教育属性，但弱化视觉噪点，突出内容。

## 2. 现状问题概览
1. **多层容器叠加**：`MainLayout` 与 `TeacherWorkspaceLayout` 双层卡片/投影造成视觉拥挤，主内容区有效宽度受限。
2. **三栏同时充满**：左右侧栏默认堆满列表、按钮，与中心区争夺注意力，信息缺乏优先级和分组。
3. **Summary 与 Footer 冗余**：首部 Summary Strip、右侧栏、底部 Footer 重复呈现“活动/公告”等信息，造成噪音。
4. **主区缺乏节奏**：筛选器、表格、大量卡片同时出现，欠缺折叠或分步组织；对话框/弹层与主体混杂。
5. **命名不统一**：仍使用“虚拟实验”表达，需统一为“实验管理”。

## 3. 全局改造策略
### 3.1 布局与容器
- 将`TeacherWorkspaceLayout`设为唯一的内容容器，`MainLayout`仅作为导航框架，去除二次白底/阴影。
- 默认采用“两栏 + 可选右栏”模式：右栏变为按需加载，优先为主内容腾出横向空间。
- Summary Strip 控制在 2-3 张卡片，保持统一高度与间距，提供“查看更多”入口而非一次性展开。
- Footer 模块改为全局浮底信息中心，按需引用；非必要页面可隐藏。

### 3.2 信息层级与组件
- 侧栏引入**标签页/手风琴**模式，仅保留当前任务最关键的过滤与上下文，辅助信息折叠或迁移到右栏。
- 中心内容采用 Section Card + 分组标题，必要时通过 Tab 或步骤条分段展示，避免单屏过载。
- 统一 Summary 指标、状态标签、操作按钮的视觉语法：色阶、圆角、阴影与动效保持一致。
- 将 AI 建议、资源引用、协作记录整合为「智能助理」组件，提供标签切换，减少重复标题。

### 3.3 响应式与交互
- 调整断点：≥1280px 显示双侧栏；960-1279px 默认折叠右栏；<960px 左右栏下移，主区占满。
- 引入全局“视图密度”开关（舒适/紧凑），配合 Element Plus 表格的 size 变体调节行高。
- 所有可折叠区提供显著的折叠指示与动效，同时兼顾键盘、读屏可访问性。

## 4. 核心页面改造方案
### 4.1 控制台（/dashboard）
- **首屏结构**：Summary Strip（本周重点 / 学情预警 / AI 建议数）+ 快速操作条。
- **主内容**：拆分为“课堂态势”“作业提醒”“教学洞察”三个 Section，通过 Tab 切换概览/详细数据。
- **右侧栏**：保留 AI 观察报告与行动建议，导出任务合并为操作按钮组。

### 4.2 课程工作台（/courses 等）
- 左侧栏：保留“分类筛选 + 快捷筛选”，最近活动迁至右侧或 Footer。
- 主区：筛选条件折叠为「筛选器抽屉」；列表视图与卡片视图使用顶部 Tab 切换，默认按照教师角色偏好的视图记忆。
- 右侧栏：整合 AI 建议、资源引用、协作记录为模块化面板，可按标签切换。

### 4.3 班级控制台（/classrooms）
- Summary Strip 聚焦“出勤率 / 互动数 / 待关注学生”。
- 主区分块：「课堂进程」、「实时互动工具」、「资源广播」；引入浮动工具条便于快速操作。
- 右侧栏保留“学生画像快照 + 家校沟通入口”，其他记录沉淀到时间轴抽屉。

### 4.4 作业管理（/assignments）
- 左侧栏仅保留课程/班级筛选与状态过滤。
- 主区按“作业列表 → 任务详情/批改台”双栏布局，批改区提供折叠式 AI 评语面板。
- Summary Strip 重点提醒“待批改数量 / 平均得分 / 异常提醒”。

### 4.5 实验管理（/labs）
- 同步改名为“实验管理”。
- 左侧栏：模板库与分组切换合并为标签；环境状态放入顶部状态条。
- 主区：步骤配置采用时间线式 Section Card，资源、评分、监考配置通过手风琴折叠呈现。
- 右侧栏：实时资源用量 + AI 脚本 + 注意事项 + 学生提交追踪采用标签切换。

### 4.6 体验管理（/experiences）
- Summary Strip 精简为「上线状态 / 反馈数 / 优化建议」。
- 主区改为「上传管理」「预览&测试」两个 Tab；埋点配置放入第二层页面或抽屉。

### 4.7 资源中心（/resources）
- 左侧保留核心筛选，标签体系采用分组折叠。
- 主区资源卡片提供密度切换；批量操作贴顶部工具条。
- 右侧栏整合推荐组合与引用统计，版权信息移动到资源详情抽屉。

### 4.8 学习分析（/analytics）
- Summary 指标拆分为 KPI + 预警数 + 上下文说明。
- 主区通过 Tab 切换“课程 / 班级 / 学生 / AI 使用”四个视角；每个视角内按卡片瀑布布局。
- 右侧栏仅保留导出与订阅配置。

## 5. 阶段性实施思路

- **短期（1-2 周）**
  - 精简 `AppSidebar` 导航，凸显课程/实验/作业/分析/AI 助手，合并低频模块；在 Dashboard 引入学科过滤器并调整 Summary 数据源。
  - 前后端打通教师学科过滤：API 支持 `teacherId + subject` 查询，前端保存偏好，默认展示“我的学科”。
- **中期（3-6 周）**
  - 构建 AI 课程工作流：课程工作台集成 AI 草拟、智能模板与版本对比；实验管理强化 Jupyter/模型训练场景，完善资源配额提示。
  - 扩展学习分析指标，使 KPI 支持 AI/IT 特色（AI 作业批改效率、模型使用频率等），以 Tab 切换多学科视角。
- **长期（6 周以上）**
  - 构建个性化 AI 教学助手：支持上下文记忆、跨课程推荐、教学策略建议。
  - 打通教育知识图谱与教学决策引擎，为跨学科场景提供推荐与预警；规划 AI 产出审计与合规体系。

## 6. 建议下一步行动

1. 输出侧边栏与导航草图，明确保留/合并/移除项，对齐 `AppSidebar.vue` 改造。
2. 落地教师学科过滤：后端接口补充 subject 参数，前端在 Dashboard、课程、实验等视图同步过滤逻辑。
3. 调整 Dashboard Summary 与面板文案，引入 AI/IT 区域及学科切换，针对 AI 工具使用、实验活跃度等指标设计组件。
4. 规划实验管理（原虚拟实验）改版原型，确保术语一致并与 AI/编程课程流程衔接。
5. 整理 AI 助手提示词/模板规范，沉淀可复用的插拔机制，服务不同学科教师。

## 7. 设计交付与协作
- **设计稿**：在 Figma 中输出桌面/平板/移动三套关键页面，附交互说明。
- **组件库**：同步 UI Kit 中的卡片、标签、Summary、侧栏组件，约束尺寸与留白。
- **研发对接**：提供布局改造任务单（MainLayout、TeacherWorkspaceLayout、各页面视图），明确拆分与验收标准。
- **效果评估**：上线后跟踪教师操作路径、页面停留时长与关键操作转化率，持续优化。
